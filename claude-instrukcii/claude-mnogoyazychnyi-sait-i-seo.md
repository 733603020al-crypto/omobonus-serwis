# Правила многоязычного сайта и SEO

Этот файл нужно читать перед любой задачей, связанной с языками, переводами, SEO, sitemap, canonical, hreflang, routes или добавлением новых языков.

---

## 1. Главный принцип сайта

Для чего этот блок:
чтобы всегда было понятно, какая версия сайта является основной.

Польская версия сайта является основной и исходной.

Корневые маршруты относятся к польской версии:

- `/`
- `/uslugi/...`
- `/kontakt`
- `/regulamin`
- `/polityka-prywatnosci`

Остальные языки являются локализациями польской структуры:

- украинский: `/uk`
- русский: `/ru`
- английский: `/en`

---

## 2. Архитектура многоязычности

Для чего этот блок:
чтобы не создавать отдельные копии сайта под каждый язык.

Нельзя дублировать дизайн, компоненты, layout и бизнес-логику для каждого языка.

Правильный подход:

- один общий дизайн;
- общие компоненты;
- общая логика;
- общая структура страниц;
- разные языковые тексты;
- разные SEO-данные;
- разные языковые URL.

Плохой подход:

- копировать польскую страницу целиком;
- создавать отдельную верстку для каждого языка;
- менять дизайн только в одной языковой версии;
- делать разные структуры страниц без согласования.

---

## 3. Что может отличаться между языками

Для чего этот блок:
чтобы понимать, что переводится, а что должно оставаться общим.

Между языковыми версиями могут отличаться:

- тексты на странице;
- тексты кнопок;
- title;
- description;
- H1;
- H2;
- FAQ;
- alt-тексты изображений, если они языковые;
- SEO-тексты;
- metadata;
- canonical URL;
- hreflang URL;
- языковые data-файлы;
- slug, если это отдельно согласовано.

Между языковыми версиями не должны отличаться без согласования:

- дизайн;
- сетка;
- порядок основных блоков;
- структура компонентов;
- логика формы;
- логика отправки заявки;
- базовые UI-компоненты.

---

## 4. Правило изменения дизайна

Для чего этот блок:
чтобы дизайн не расходился между языковыми версиями.

Если меняется дизайн, layout, структура блока или компонент в польской версии, такое же структурное изменение должно быть применено ко всем существующим языковым версиям.

Пример:

если изменился блок hero на польской главной странице, нужно проверить соответствующие версии:

- `/uk`
- `/ru`, если уже существует;
- `/en`, если уже существует.

---

## 5. Canonical

Для чего этот блок:
чтобы Google понимал основной адрес каждой конкретной языковой страницы.

Каждая реальная языковая страница должна иметь self-canonical.

Пример:

- `/uslugi/serwis-laptopow` canonical указывает на `/uslugi/serwis-laptopow`;
- `/uk/uslugi/serwis-laptopow` canonical указывает на `/uk/uslugi/serwis-laptopow`;
- `/ru/uslugi/serwis-laptopow` canonical указывает на `/ru/uslugi/serwis-laptopow`;
- `/en/uslugi/serwis-laptopow` canonical указывает на `/en/uslugi/serwis-laptopow`.

Нельзя делать так, чтобы все языковые версии указывали canonical только на польскую страницу.

---

## 6. Hreflang

Для чего этот блок:
чтобы Google понимал, что разные URL являются версиями одной страницы на разных языках.

Каждая реальная языковая страница должна иметь hreflang на все существующие языковые версии этой же страницы.

Для главной страницы:

- `pl` ведет на `/`;
- `uk` ведет на `/uk`;
- `ru` ведет на `/ru`, только если страница реально существует;
- `en` ведет на `/en`, только если страница реально существует;
- `x-default` ведет на польскую версию `/`.

Нельзя добавлять hreflang на URL, которых нет.

---

## 7. HTML lang

Для чего этот блок:
чтобы браузер и поисковики понимали язык страницы.

Каждая языковая версия должна иметь правильный язык HTML:

- польская версия: `pl`;
- украинская версия: `uk`;
- русская версия: `ru`;
- английская версия: `en`.

---

## 8. Sitemap

Для чего этот блок:
чтобы поисковики видели все реальные страницы сайта.

В sitemap должны попадать только реальные существующие страницы.

Если страницы `/ru/...` или `/en/...` еще не созданы, их нельзя добавлять в sitemap.

При добавлении нового языка нужно проверить:

- главную страницу языка;
- страницы услуг;
- контакт;
- юридические страницы, если они реально существуют на этом языке;
- canonical;
- hreflang.

---

## 9. Индексация и HTML

Для чего этот блок:
чтобы основной SEO-контент был виден Google сразу.

Google должен видеть основной текст страницы сразу в HTML.

В HTML должны быть доступны:

- title;
- description;
- H1;
- основной текст страницы;
- основные тексты услуг;
- важные SEO-блоки;
- FAQ, если он используется для SEO.

Нельзя делать так, чтобы важный SEO-текст появлялся только после client-side JavaScript.

Динамические блоки допустимы, если они не являются основным SEO-контентом.

Например, Google Reviews может грузиться динамически, если основные SEO-тексты страницы уже есть в HTML.

---

## 10. Добавление новых языков

Для чего этот блок:
чтобы русский и английский добавлялись правильно, а не копированием страниц.

Перед добавлением нового языка нужно сначала проверить текущие польскую и украинскую версии:

- routes;
- layout;
- page metadata;
- canonical;
- hreflang;
- sitemap;
- html lang;
- статический HTML для title / description / H1 / основного текста.

При добавлении нового языка нужно добавить:

- языковые маршруты;
- переводы;
- service data;
- metadata;
- title;
- description;
- H1;
- sitemap entries;
- canonical;
- hreflang;
- html lang.

Нельзя добавлять новый язык через копирование всей верстки.

---

## 11. Перед изменениями

Для чего этот блок:
чтобы сначала был план, а не хаотичные правки.

Перед изменениями нужно сначала дать короткий план:

- какие файлы будут проверены;
- какие файлы будут изменены;
- что именно будет изменено;
- как проверить результат.

Не менять без подтверждения:

- тексты;
- цены;
- контакты;
- SEO;
- юридические страницы;
- структуру URL;
- sitemap;
- canonical;
- hreflang.

---

## 12. Текущее состояние сайта

Для чего этот блок:
чтобы зафиксировать уже найденную информацию и не проверять одно и то же заново без необходимости.

На момент проверки известно:

Что OK:

- есть структура языковых URL;
- польская версия находится в корне сайта;
- украинская версия находится в `/uk`;
- есть lang-атрибуты;
- есть canonical;
- есть hreflang на внутренних страницах;
- есть sitemap;
- есть robots.txt;
- title / description / H1 отдаются в статическом HTML.

Что нужно проверить или исправить:

- hreflang на главных страницах `/` и `/uk`;
- hreflang для `/polityka-prywatnosci`;
- hreflang для `/regulamin`;
- не добавлять украинские ссылки для юридических страниц, если украинских страниц реально нет.

Google Reviews не является главным SEO-блоком и может грузиться динамически.

Файлы, связанные с мультиязычностью:

- `src/lib/i18n/uk.ts`;
- `src/lib/services-data.ts`;
- `src/lib/services-data-uk.ts`;
- `src/app/(pl)/layout.tsx`;
- `src/app/uk/layout.tsx`.

Файлы, связанные с SEO:

- `page.tsx`;
- `layout.tsx`;
- `src/app/sitemap.ts`;
- `public/robots.txt`.
